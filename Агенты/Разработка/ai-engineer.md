## Стек
- **YOLOv8** (Ultralytics) — локальный self-hosted
- **PyTorch** + **ONNX Runtime** (для inference)
- **Google Gemini 2.0 Flash API** — fallback, multimodal
- **Anthropic Claude Sonnet** — AI-диетолог (чат)
- **LangChain** — LLM оркестрация
- **Sentence Transformers** — embeddings для семантического поиска продуктов
- **TorchServe** или **BentoML** — для production inference

## Архитектура распознавания еды

```
Фото от пользователя
    ↓
YOLOv8 (локально, быстро, бесплатно)
    ↓
Confidence >= 0.7?
    ↓ НЕТ                    ↓ ДА
Gemini Vision API          Вернуть результат
(точнее, $0.001/запрос)
    ↓
Вернуть результат
```

**Почему так:** YOLO бесплатный и быстрый (<500ms), но плохо на редких блюдах. Gemini дороже, но точнее.

## Задача 1: Дообучение YOLOv8 на русской еде

### Датасет (нужно собрать)
- 500-1000 фото на класс × 30 классов = 15-30К фото
- Классы: борщ, блины, пельмени, творог, гречка, куриная грудка, рис, яйца, протеиновые коктейли, и т.д.
- Источники: 
  - Open Food Facts (с фото)
  - Crowdsourcing (просить юзеров отмечать)
  - Стоки (Freepik, Unsplash)
  - Своя фотосессия (бюджетный вариант)

### Обучение
```bash
yolo detect train \
  data=gymfuel_food.yaml \
  model=yolov8n.pt \
  epochs=100 \
  imgsz=640 \
  batch=32 \
  device=cuda
```

### Метрики успеха
- **mAP50 ≥ 0.87** на валидационном датасете
- **Inference time <500ms** на CPU
- **Модель <50MB** (для быстрой загрузки)

## Задача 2: Gemini Vision fallback

### Промпт для Gemini
```python
FOOD_RECOGNITION_PROMPT = """
Ты — эксперт по распознаванию еды. На фото тарелка с едой.

Определи:
1. Какие продукты на фото (список на русском)
2. Примерный вес каждого в граммах
3. КБЖУ каждого (на указанный вес)

Формат ответа — строго JSON:
{
  "items": [
    {
      "name_ru": "куриная грудка",
      "grams": 200,
      "kcal": 330,
      "protein": 62,
      "carbs": 0,
      "fat": 7
    }
  ],
  "confidence": 0.85
}

Если не уверен — укажи confidence < 0.7.
Если еды нет на фото — верни {"items": [], "confidence": 0.0}.
"""
```

### Экономия
- Кеш результатов по хешу фото (SHA256)
- Batch-запросы через Gemini Batch API
- Даунсайз фото до 512x512 перед отправкой

## Задача 3: AI-диетолог (чат с Claude)

### System prompt
```
Ты — AI-диетолог GymFuel AI. Твоя задача — помочь качку добивать норму белка и КБЖУ.

Контекст о пользователе:
- Цель: {goal} (масса / сушка / поддержка)
- Норма: {calories} ккал, {protein}г белка
- Съел сегодня: {consumed_summary}
- Спортпит на полке: {supplements}

Правила:
1. Отвечай кратко (2-3 предложения)
2. Tone: прямой, без розовых соплей. Можно с провокацией.
3. Всегда конкретные цифры и продукты
4. Если пользователь просит план — давай таблицу
5. Никогда не давай медицинских советов

Примеры стиля:
- "Не добрал 40г белка. 1 ложка Whey = 30г. Остальное — 150г творога."
- "На сушке ешь курицу/рыбу, не фарш. Жира меньше."
```

### RAG для истории пользователя
- Supabase + pgvector
- Embedding дневника последних 7 дней
- Поиск похожих дней для контекста

## Стоимость (оценка)

| Модель | Цена | На 1000 распознаваний | На 10K MAU |
|--------|------|----------------------|------------|
| YOLOv8 (self-hosted) | $0 | $0 | ~$50/мес инфра |
| Gemini 2.0 Flash | $0.001/img | $1 | ~$300/мес |
| Claude Sonnet (чат) | $3/M input | ~$50 | ~$500/мес |

**Оптимизация:**
- Кешировать фото → экономия 20-30%
- Prompt caching для system prompt в Claude → экономия 50% на input
- Gemini Batch вместо real-time → экономия 50%
