## 1. ЦЕЛИ И ЗАДАЧИ

### 1.1. Цель проекта
Создать веб-приложение (PWA), которое помогает людям в тренажёрном зале:
1. Фиксировать дневной рацион (фото + текст + голос)
2. Считать КБЖУ под цель (масса / сушка / поддержка)
3. Автоматически встраивать спортивное питание в план дня
4. Получать обратную связь от AI-диетолога

### 1.2. Задачи MVP
- Реализовать регистрацию и профиль с расчётом норм КБЖУ
- Распознавать еду по фото (с ручным override)
- Считать потреблённые КБЖУ за день
- Дать калькулятор спортпита (базовые добавки)
- Сделать дневник питания с разбивкой по приёмам

### 1.3. Критерии успеха MVP
- Retention 40%+ на 7-й день
- Среднее время в приложении 5+ мин/день
- 50+ пользователей в закрытом бета-тесте
- NPS ≥ 30

## 2. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

### 2.1. Модуль: Авторизация и профиль

**F-2.1.1. Регистрация**
- Email + пароль
- OAuth: Google, Apple
- Email-верификация обязательна

**F-2.1.2. Онбординг (5 шагов)**
1. Пол, возраст
2. Рост, вес, % жира (опционально)
3. Цель: "Набор массы" / "Сушка" / "Поддержка"
4. Уровень активности: "2-3 тренировки" / "4-5" / "6+"
5. Какой спортпит используешь (чек-лист: Whey, Казеин, Креатин, BCAA, Гейнер, Пред-тренировочник)

**F-2.1.3. Автоматический расчёт норм**
При сохранении профиля считать:
- **BMR** (Mifflin-St Jeor):
  - М: `10*вес + 6.25*рост − 5*возраст + 5`
  - Ж: `10*вес + 6.25*рост − 5*возраст − 161`
- **TDEE:** BMR × коэффициент активности (1.375 / 1.55 / 1.725)
- **Норма калорий:**
  - Масса: TDEE × 1.15
  - Сушка: TDEE × 0.80
  - Поддержка: TDEE × 1.0
- **Белок:** 2.0 г × вес (масса), 2.3 г × вес (сушка), 1.8 г × вес (поддержка)
- **Жиры:** 0.8-1.0 г × вес
- **Углеводы:** остаток от калорий (1 г белка = 4 ккал, 1 г углеводов = 4 ккал, 1 г жиров = 9 ккал)

**F-2.1.4. Редактирование профиля**
Можно изменить вес, цель, спортпит в любой момент. После изменения — пересчёт норм.

### 2.2. Модуль: Дневник питания

**F-2.2.1. Структура дня**
Приёмы пищи (кастомизируемые):
- Завтрак
- Перекус 1
- Обед
- Перекус 2
- До тренировки
- После тренировки
- Ужин
- Перед сном

**F-2.2.2. Добавление еды — 3 способа**

**Способ 1: По фото**
1. Пользователь нажимает "📸 Фото"
2. Открывается камера или загрузка файла
3. Фото отправляется в бэкенд → YOLOv8 → Gemini Vision (fallback)
4. Возвращается список распознанных продуктов с граммовкой
5. Пользователь подтверждает / редактирует / добавляет
6. Сохраняется в дневник

**Способ 2: Ручной поиск**
1. Пользователь вводит название ("куриная грудка")
2. Автокомплит из базы (USDA + Open Food Facts + локальная)
3. Выбирает продукт
4. Указывает граммовку
5. Сохраняется

**Способ 3: Голосовой ввод (v1.0, не MVP)**
"съел 200 грамм куриной грудки и 150 риса"

**F-2.2.3. Отображение дня**
Верхняя панель:
- Прогресс-бары: Калории / Белки / Жиры / Углеводы (съедено / норма)
- Цветовая индикация: зелёный (в норме), жёлтый (80-120%), красный (перебор)

Список приёмов пищи с развёртыванием:
- Название продукта, граммовка, КБЖУ
- Кнопки: редактировать, удалить, дублировать

**F-2.2.4. Календарь**
Навигация по дням назад (история).
На MVP — 30 дней истории для FREE, без лимита для PRO.

### 2.3. Модуль: Калькулятор спортпита

**F-2.3.1. База спортпита (встроенная)**
Минимум 20 популярных продуктов с полным КБЖУ и рекомендациями:

| Продукт | Пример бренда | КБЖУ на порцию | Когда пить |
|---------|---------------|----------------|------------|
| Whey Protein | Optimum Nutrition 100% Whey | 30г белка / 120 ккал | После тренировки / между приёмами |
| Казеин | MyProtein Casein | 25г белка / 110 ккал | Перед сном |
| Гейнер | Mutant Mass | 50г белка / 1000 ккал | После тренировки (на массе) |
| Креатин моногидрат | Prime Kraft | 0 ккал | Утром, 5г/день |
| BCAA | Scitec BCAA | 7г / 30 ккал | Во время тренировки |
| Пред-тренировочник | C4 | Варьируется | За 30 мин до тренировки |

**F-2.3.2. Калькулятор (MVP)**
Логика:
1. Смотрим норму белка пользователя и сколько уже съедено
2. Если **разница > 30г** — предлагаем порцию Whey ("Добавь 1 ложку протеина — закроет 30г белка")
3. На массе — если осталось много углеводов → предлагаем гейнер или банан + протеин
4. Всегда (в тренировочный день) — напоминание о креатине (5г утром)

**F-2.3.3. Интеграция в дневник**
Спортпит добавляется как обычный продукт в нужный приём пищи.
Отдельный бейдж "💊 Спортпит" для визуального разделения.

### 2.4. Модуль: AI-диетолог (v1.0, не MVP)

Чат с AI, который видит контекст пользователя:
- Профиль (цель, нормы)
- История питания за 7 дней
- Запас спортпита на полке

Примеры запросов:
- "Что мне съесть на завтрак?"
- "Я не добираю 50г белка — как быстро их закрыть?"
- "Составь план на день ног"

Модель: **Claude Sonnet** (через API) с system prompt и RAG по рациону пользователя.

### 4.2. Потоки данных

**Распознавание фото:**
```
User → Upload photo → Next.js /api/recognize
  → POST backend /recognize
  → 1. YOLOv8 (локально, быстро, для известных блюд)
  → 2. Если confidence <0.7 → Gemini Vision (точнее, но дороже)
  → 3. Вернуть [{name, grams, kcal, protein, carbs, fat}]
  → Показать пользователю для подтверждения
  → Сохранить в БД (meals table)
```

**AI-диетолог:**
```
User пишет в чат → Next.js /api/chat (streaming)
  → Backend: собрать контекст (профиль + история 7 дней + спортпит)
  → Claude Sonnet через API (system prompt + context + user message)
  → Stream ответ обратно через WebSocket
```

```sql
id              UUID PRIMARY KEY
email           TEXT UNIQUE
created_at      TIMESTAMP
subscription    TEXT -- 'free' | 'pro' | 'pro_plus'
```

**profiles**
```sql
id              UUID PRIMARY KEY

gender          TEXT -- 'male' | 'female'

goal            TEXT -- 'bulk' | 'cut' | 'maintain'

tdee            INT    -- рассчитанный
target_kcal     INT    -- рассчитанный
target_protein  INT
target_carbs    INT
target_fat      INT
supplements     JSONB  -- ['whey', 'creatine', 'bcaa']
updated_at      TIMESTAMP
```

**foods** (справочник продуктов)
```sql
id              UUID PRIMARY KEY
external_id     TEXT     -- id из USDA или OFF
source          TEXT     -- 'usda' | 'off' | 'custom' | 'supplement'
name_ru         TEXT
name_en         TEXT
kcal_100g       NUMERIC
protein_100g    NUMERIC
carbs_100g      NUMERIC
fat_100g        NUMERIC
embedding       VECTOR(384)  -- для семантического поиска
created_at      TIMESTAMP
```

**meals** (дневник)
```sql
id              UUID PRIMARY KEY
user_id         UUID REFERENCES users(id) ON DELETE CASCADE
date            DATE
meal_type       TEXT  -- 'breakfast' | 'lunch' | 'pre_workout' | ...
food_id         UUID REFERENCES foods(id)
grams           NUMERIC
kcal            NUMERIC  -- вычислено
protein         NUMERIC
carbs           NUMERIC
fat             NUMERIC
photo_url       TEXT NULLABLE
source          TEXT  -- 'photo' | 'search' | 'voice'
created_at      TIMESTAMP
```

**supplements_stock** (что у пользователя на полке)
```sql
id              UUID PRIMARY KEY
user_id         UUID REFERENCES users(id)
supplement_type TEXT  -- 'whey' | 'creatine' | ...
brand           TEXT
serving_size_g  NUMERIC
protein_per_serving NUMERIC
kcal_per_serving NUMERIC
servings_left   INT
added_at        TIMESTAMP
```

**weight_logs**
```sql
id              UUID PRIMARY KEY
user_id         UUID
weight_kg       NUMERIC(5,2)
logged_at       DATE
```

**chat_messages** (для AI-диетолога)
```sql
id              UUID PRIMARY KEY
user_id         UUID
role            TEXT  -- 'user' | 'assistant'
content         TEXT
created_at      TIMESTAMP
```

### 5.2. Row-Level Security (RLS)
Все таблицы с `user_id`:
```sql
CREATE POLICY "Users see own data" ON <table>
  FOR ALL USING (auth.uid() = user_id);
```

## 6. API ENDPOINTS (Backend FastAPI)

### Auth (через Supabase, не дублируем)

### Profile
- `GET /api/profile` — текущий профиль
- `POST /api/profile` — создать/обновить
- `POST /api/profile/recalculate` — пересчитать нормы

### Food Recognition
- `POST /api/recognize/photo` — `multipart/form-data` с фото → список продуктов
- `POST /api/recognize/voice` — аудио → список продуктов (v1.0)
- `POST /api/recognize/barcode` — баркод → продукт (v1.0)

### Meals (дневник)
- `GET /api/meals?date=2026-04-21` — приёмы за день
- `POST /api/meals` — добавить приём
- `PATCH /api/meals/{id}` — обновить
- `DELETE /api/meals/{id}`
- `GET /api/meals/stats?from=YYYY-MM-DD&to=YYYY-MM-DD`

### Foods (поиск)
- `GET /api/foods/search?q=курица&limit=20` — автокомплит
- `GET /api/foods/{id}` — детали

### Supplements
- `GET /api/supplements/stock` — мой запас
- `POST /api/supplements/stock` — добавить
- `GET /api/supplements/recommendations` — что пить сегодня

### AI Chat (v1.0)
- `POST /api/chat` — streaming через SSE/WebSocket

### Progress
- `POST /api/weight` — записать вес
- `GET /api/progress?period=30d`
