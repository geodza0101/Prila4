## Часть 3. Требования к технической реализации (для разработчиков)

3.1. Каждое согласие сохраняется в таблице `user_consents` со следующими полями:
```
id UUID PRIMARY KEY
user_id UUID REFERENCES users(id)
consent_type TEXT -- 'general' | 'special_categories' | 'cross_border' | 'marketing'
consent_version TEXT -- '1.0'
granted BOOLEAN
granted_at TIMESTAMP
ip_address INET
user_agent TEXT
revoked_at TIMESTAMP NULLABLE
policy_hash TEXT -- хеш текста политики на момент согласия
```

3.2. При изменении версии документов (Privacy Policy, Terms) пользователь должен заново акцептовать изменения.

3.3. Отзыв согласия = запись `revoked_at`, а не удаление записи (для целей доказывания).

3.4. Логи согласий должны экспортироваться по запросу Пользователя (право на переносимость) и по запросу Роскомнадзора.

3.5. Чек-боксы **не должны** быть предварительно отмечены (pre-ticked) — это нарушение GDPR (дело C-673/17 Planet49).

3.6. Текст согласия должен быть доступен для печати и скачивания в PDF.

3.7. Кнопки «Принять всё» без возможности раздельного выбора — запрещены.
