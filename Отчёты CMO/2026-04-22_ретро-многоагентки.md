## 🎬 Хронология работы по этапам

### Этап 1: Концепция (до агентов)
**Длительность:** ~40 мин
- Исследование рынка (через `researcher` agent)
- Написание `Идея.md` (продуктовая концепция)
- Написание `Техническое задание.md` (функциональное ТЗ)

**Файлов:** 3 | **Размер:** 59 KB

### Этап 2: Дизайн-пайплайн (6 агентов последовательно + параллельно)
**Длительность:** ~25 мин

Пайплайн:
```
Агент 1 (researcher) → собрал 58+ ссылок конкурентов
        ↓
Агент 2 (researcher) → проанализировал дизайн, выводы по TOP-10
        ↓
Агент 3 (general) ─ ТЗ приложения ┐ (параллельно)
Агент 5 (general) ─ ТЗ лендинга   ┘
        ↓
Я собрал финальный «Требование к дизайну.md»
```

**Результаты:**
- 58+ ссылок на конкурентов
- Глубокий анализ дизайна 10 конкурентов
- 2 больших ТЗ (приложение 62 KB + лендинг 67 KB)
- 1 финальный сжатый документ для дизайнера (21 KB)

### Этап 3: Создание многоагентной команды
**Длительность:** ~15 мин

Создал:
- 14 файлов агентов (в `/Агенты/Маркетинг`, `/Разработка`, `/Связующие`)
- Скопировано в `~/.claude/agents/` для глобальной активации
- `README.md` + обсидиан-визуализация с графом связей

Потом добавил ещё 3 агента (Экспертиза) + 2 системных = 17 итого.

### Этап 4: Первый боевой спринт — 7 агентов параллельно
**Длительность:** ~30 мин (параллельная работа)
**Это главное событие сессии**

Запустил одним сообщением **7 Agent tool вызовов параллельно:**

| Агент | Задача | Результат |
|-------|--------|-----------|
| 🔎 **market-researcher** | Боли ЦА из Reddit/VC/App Store | 25+ цитат с источниками |
| ✍️ **copywriter** | Тексты лендинга, постов, email | 5 файлов (лендинг + TG + email + ASO) |
| 🎬 **warmup-content** | 14-дневный прогрев TG | 14 готовых постов |
| 🎨 **ui-ux-designer** | Прототипы экранов | Dashboard.tsx + PhotoRecognition.tsx + preview.html |
| 🥗 **nutrition-expert** | Валидация формул КБЖУ | ⚠️ Нашёл 3 критические ошибки |
| ⚖️ **legal-compliance** | Privacy Policy, Terms, ПДн | 5 юр.документов (129 KB) |
| 🔧 **backend-dev** | Скелет FastAPI | 19 файлов (код + миграция БД + тесты) |

**Итог параллельной работы:**
- **36 файлов** создано за ~30 минут
- **~670K токенов** суммарно использовано
- Результаты собраны в единый CMO-отчёт через skill `cmo-team-report`

### Этап 5: Реальная веб-разработка (сам, не через агентов)
**Длительность:** ~45 мин
**Почему не через агентов:** итеративная работа с быстрой обратной связью от пользователя — агенты добавили бы overhead.

- Создал 3 HTML-страницы: `index.html` (лендинг), `register.html`, `app.html`
- **4 итерации деплоя на Vercel:**
  1. MVP-лендинг
  2. Премиум-редизайн (grain, gradient mesh, Linear/Stripe вайб)
  3. Интеграция OpenRouter (распознавание еды по фото)
  4. iPhone-рамка на десктопе + Дневник + Спортпит + AI-диетолог

Использовал `researcher`, `warmup-content` и `presentation-designer` из существующих глобальных агентов — **не создавал новых там, где было старое решение.**

## 📈 Что сработало отлично

### 1. ✅ **Параллельный запуск 7 агентов**
Вместо 7 × 5 мин = 35 минут последовательно — получилось **30 минут параллельно** для всех сразу. Это главное преимущество многоагентной системы в Claude Code.

### 2. ✅ **Роль оркестраторов**
`cmo-orchestrator` и `cto-orchestrator` чётко развели ответственности. Когда нужно было делегировать — сразу понятно куда. Нет дублирования работы.

### 3. ✅ **Экспертиза как отдельный отдел**
`nutrition-expert` **нашёл 3 критические ошибки в формулах КБЖУ** из ТЗ, которые в массовой разработке привели бы к юридическим рискам (калории <1500 для мужчин — небезопасно). Без него их бы не заметили.

### 4. ✅ **Skill `cmo-team-report`**
Написанный кастомный skill сразу появился в системе и позволил собирать отчёты от субагентов в один файл. Георгий получил понятный CMO-отчёт на 5 пунктов TL;DR вместо разбросанных результатов.

### 5. ✅ **Obsidian-визуализация**
Граф из `🏢 Карта агентов.md` + `🎯 Карта скилов.md` позволяет видеть связи. Правило `obsidian-agents-sync.md` автоматически обновляет структуру при любых изменениях.

### 6. ✅ **Переиспользование глобальных агентов**
Не дублировал — использовал готовых: `researcher`, `marketer`, `warmup-content`, `sales-agent`. Это экономит время создания и даёт проверенное качество.

## ⚠️ Что не сработало / узкие места

### 1. 🟡 **Агенты не сохраняют файлы сами**
Первые 2 research-агента вернули контент, но **не записали в файл**. Пришлось сохранять вручную после получения ответа. Решение — явно прописывать в промпте «сохрани через Write», но это не всегда срабатывает.

### 2. 🟡 **Агенты ≠ одна сессия Claude Code**
Созданные агенты активируются только в **новой сессии** Claude Code. В текущей — они видны через `Agent tool`, но не через `/agents` command.

### 3. 🟡 **Ключ API OpenRouter захардкожен в клиенте**
Для демо — ок, но в продакшене это уязвимость. Нужен бэкенд-прокси (backend-dev уже написал скелет FastAPI — осталось только поднять).

### 4. 🟡 **Модель `gemini-2.0-flash-exp:free` исчезла**
OpenRouter удалил бесплатную Gemini — пришлось переключаться на платную `gemini-2.5-flash-lite`. **Урок:** полагаться на бесплатные API рискованно, всегда нужен fallback.

### 5. 🟡 **Некоторые агенты не проверили промпт на security**
Первая версия `app.html` использовала `innerHTML` для рендеринга AI-ответов — security hook заблокировал запись. Пришлось переписать через `createElement + textContent`. Агенты не защищают от XSS автоматически.

## 🧠 Ключевые инсайты

### 1. Оркестраторы > исполнители
Без `cmo-orchestrator` и `cto-orchestrator` пришлось бы каждого агента вручную запускать. С ними — одна команда «Собери отчёт команды» разворачивается в 4 параллельных запроса.

### 2. Экспертиза экономит деньги
`nutrition-expert` ловит ошибки за секунды, которые стоили бы штрафов или отзывов ЦА через месяцы. Один специализированный агент за 3 минуты сделал ревью, за которое нутрициолог берёт 20K ₽.

### 3. Чем уже роль агента — тем лучше результат
`copywriter` пишет только тексты — пишет хорошо. Общий `marketer` размазывался бы между стратегией и копирайтом.

### 4. Параллельность — главное преимущество
В последовательной работе 1 человек = 1 задача. В многоагентной — 7 задач одновременно.

### 5. Skills > ручные промпты
Кастомный skill `cmo-team-report` экономит 5 минут на каждом отчёте, т.к. формат и логика уже прописаны.
