### Целевая аудитория (сегменты)
| Сегмент | Доля | Особенности |

1. **Неточные КБЖУ** — не учитывают индивидуальные особенности метаболизма
2. **Сложный ввод** — ручной поиск продукта долгий и скучный
3. **Отсутствие мотивирующего AI** — унылые статичные советы
4. **Статичные дневные нормы** — не адаптируются к прогрессу
5. **Плохое покрытие русской кухни** — нет борща, блинов, рассольника с правильным КБЖУ
6. **MyFitnessPal стагнирует** — продана за $345M (ниже цены покупки)

## 3. AI-ТЕХНОЛОГИИ

### Computer Vision для еды

**1. YOLOv8/v11/v12** (рекомендую для MVP)
- Точность: 85-95% на известные категории
- mAP: 0.873 на датасете 1300 блюд
- Скорость: один проход, все объекты одновременно
- Пример вывода: пицца → 317 ккал (10g белки, 40g углеводы, 13g жиры)

**2. CLIP (OpenAI)**
- Связывает изображение с текстом
- Хорошая генерализация
- Медленнее YOLO

**3. Кастомные CNN**
- Точность: 90-95% при хороших данных
- Требует 500-1000+ примеров на класс

### Portion Estimation (оценка размера порции)

**1. Depth Sensing** (SnapCalorie)
- Точность: до 93%
- Железо: IR-датчики (iPhone Pro с LiDAR, Kinect)
- Минус: не работает на отражающих/прозрачных поверхностях

**2. 3D Object Scaling** (рекомендую)
- Reference objects: монета, рука, тарелка стандартного размера
- Deep Learning восстанавливает 3D из 2D
- Точность: 88-93%
- **Работает на любом смартфоне** — нужен для веб-приложения

**3. Fusion approach**
- Семантические + геометрические фичи
- Точность: 95%+

### Мультимодальные LLM (цены 2026)

| Модель | Цена | Применение | Качество |
|--------|------|------------|----------|
| **GPT-4V** | $10-20/1000 calls | Анализ фото + текст | Высокое |

## 4. КЛЮЧЕВЫЕ ФИЧИ ПО ЭТАПАМ

### MVP (обязательный минимум)
1. Распознавание еды по фото (YOLOv8)
2. Ручной ввод продуктов + поиск (USDA API)

5. Визуализация прогресса (графики)
6. Auth + облачная синхронизация (Supabase)

### v1.0 (конкурентоспособный продукт)
+ Голосовой ввод ("съел борщ и котлету")

+ Адаптивные цели (на основе прогресса)
+ Калькулятор рецептов

### v2.0 (уникальное УТП)
+ Интеграция Whoop / Apple Watch / OURA
+ Персональный AI-тренер (коучинг)
+ Генерация рецептов по продуктам в холодильнике
+ Community + челленджи
+ B2B (корпоративное здоровье, фитнес-студии)

## 5. TECH STACK

### Frontend: **Next.js 15 + React 19**
- Server Components для скорости
- API Routes встроены
- Image Optimization (важно для фото еды)
- **PWA** для мобильного веба (установка как нативное)
- shadcn/ui + Tailwind CSS
- SEO-friendly

### Backend: **FastAPI + Python 3.11**
- Async/Await (100+ конкурентных запросов)
- Pydantic для валидации
- Auto-docs (Swagger)
- WebSocket для real-time чата с диетологом
- LangChain для LLM-оркестрации

### Database: **Supabase (PostgreSQL)**
- Row-Level Security (GDPR-ready)
- Vector DB (pgvector) для семантического поиска по продуктам
- Real-time subscriptions
- Встроенная auth

### Auth: **Clerk** или Supabase Auth
- Email/Password + Google/Apple OAuth
- Webhooks для синхронизации

### AI/LLM: **LangChain + Gemini 2.0**
- Самый дешёвый мультимодальный API
- Альтернатива: OpenAI / Anthropic при росте

### Computer Vision: **YOLOv8** (TorchServe или ONNX Runtime)

### Deployment
- **Frontend:** Vercel
- **Backend:** Railway
- **Database:** Supabase

## 7. ЮРИДИКА

### GDPR (если есть европейцы)
- Мед-данные = специальная категория (повышенная защита)
- DPO нужен при 250+ пользователях
- Explicit consent (активное согласие)
- Right to be forgotten (удаление данных)
- Data portability (экспорт JSON)

### Российское 152-ФЗ
- Privacy Policy на русском
- Согласие на обработку ПДн
- Локальные серверы ИЛИ Supabase EU (допускается)

### Медицинская лицензия
- **НЕ нужна** если: не ставите диагнозы, не лечите, только информирование
- **Дисклеймер:** "Это не медицинское устройство. Консультируйтесь с врачом."

### Решение
Supabase (EU) + Privacy Policy на русском + Row-Level Security + HTTPS + Disclaimer

## 10. РИСКИ И PRE-VALIDATION

### Главные риски

| Риск | Вероятность | Влияние | Решение |
|------|-------------|---------|---------|
| Неточность AI | Высокая | Высокое | Редактирование + crowd-source |
| Низкий retention | Высокая | Критическое | Персонализация + гейминг + push |
| Конкуренция MFP | Очень высокая | Высокое | Узкое нишевание + B2B |
| LLM costs | Средняя | Среднее | Кеширование, Gemini, batch |
| GDPR compliance | Средняя | Среднее | Supabase EU + RLS |

### Pre-validation (1-2 недели до кода)

1. **Лендинг + опрос** 20 людей про боли MyFitnessPal
2. **Google Forms** для 100+ целевой аудитории: "Платили бы $10/мес?"
3. **Figma-прототип**, 50 пользователей тестируют
4. **Если 60%+ хотят AI-диетолога** → зелёный свет

### Micro-cohort (50 бета-пользователей)
- NPS >30 и retention >50% на неделю → продолжать
- NPS <30 → переделать value proposition

## 🎯 TL;DR — ГЛАВНЫЕ ВЫВОДЫ

1. **Рынок переполнен, но AI создаёт окно** — MyFitnessPal стагнирует, у новичков есть шанс
2. **Главная боль:** неточные КБЖУ, скучный UX, плохая русская кухня
3. **Tech:** Next.js + FastAPI + YOLOv8 + Gemini ($300-500/мес MVP)
4. **Монетизация:** Freemium $9.99/мес, 4% конверсия → $40K MRR на 100K MAU
5. **MVP:** 1-2 месяца, цель retention 40% день 7
6. **Выделение:** Русский рынок + Vibe-AI коуч + B2B интеграции
7. **Рекомендация:** ✅ ДА, но сначала 1-2 недели валидации (опросы, лендинг, Figma)
8. **Стоп-сигнал:** если <60% ЦА готовы платить — переделать оффер
9. **Главный конкурент:** Kaloria (РФ) и Nutrola (мировой) — следить пристально
10. **Первый шаг:** не писать код, а собрать 100 ответов от ЦА
