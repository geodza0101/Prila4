## 0. Контекст для дизайнера

Это **маркетинговый лендинг**, а не приложение. Его единственная задача — за 10-15 секунд объяснить посетителю продукт, закрыть возражения и подвести к кнопке "Скачать". Лендинг живёт на отдельном домене и может иметь дизайн, отличающийся от самого приложения.

**Само приложение GymFuel AI — тёмное (#0F0F0F, оранжевый акцент).** Лендинг может быть другим — ниже выбран вариант, который даёт максимальную конверсию.

## 1. Цели лендинга

### 1.1 Основная KPI
**Конверсия в установку приложения / регистрацию / подписку на ранний доступ.**

- Celевой показатель: **3.5-5%** от уникальных посетителей (в индустрии fitness-app — 2-4%, мы целимся выше за счёт узкой ЦА)
- Основной CTA: `[Скачать бесплатно]` → App Store / Google Play
- Вторичный CTA: `[Попробовать в вебе]` → PWA-версия

### 1.2 Вторичные KPI

| Метрика | Значение | Как измеряем |
|---|---|---|
| Сбор email на ранний доступ (до релиза) | 15-20% | Plausible + кастомный endpoint |
| Brand awareness (уникальные визиты) | 50K за 3 мес | Plausible |
| Средний scroll depth | >60% | PostHog |
| Время на странице | >45 сек | Plausible |
| Возврат (returning) | >20% | Plausible cookie-less |
| Клик по сравнительной таблице | >8% | PostHog event |

### 1.3 Воронка
```
Реклама (TG Ads / VK / Яндекс) → Лендинг →
  → Hero CTA (60%) ─┐
  → Scroll → Features → Pricing → CTA (30%) ─┼─→ App Store / Google Play
  → Email capture в модалке выхода (10%) ─┘
```

### 2.1 Портрет посетителя
**Антон, 28 лет, Москва. Сисадмин. Тренируется 4 раза в неделю, масса.** Только что искал в Google "приложение для подсчёта белка" или "приложение для набора массы". Увидел креатив "Не добираешь белок? AI посчитает за тебя" в TG Ads, кликнул. Теперь он здесь.

## 3. Design Principles лендинга (7 принципов)

1. **Mobile-first, без компромиссов.** 70% трафика с телефона. Десктоп — бонус, не основа.
2. **Max 3 секции выше fold'а на мобиле.** Hero → Social Proof → первая Feature. Всё остальное — скролл.
3. **Social proof везде, где можно дышать.** Цифры, рейтинги, отзывы, логотипы брендов — в каждой 2-3 секции.
4. **CTA дублируется минимум 5 раз.** Hero, после features, после сравнения, после pricing, final CTA. Пользователь не должен искать кнопку.
5. **Цифры крупнее слов.** "1800 ккал", "145г белка", "10K+ качков", "85%" — выделены огромным шрифтом. Мозг качка любит числа.
6. **Провокация > вежливость.** Заголовки жёсткие, прямые, без мотивационного сиропа. "Хватит гадать" > "Начните свой путь".
7. **Скорость — фича.** Лендинг грузится <2с. LCP <2.5с. Никакого Webflow-кода, никаких тяжёлых видео в hero.

## 4. Дизайн-система лендинга

### 4.1 Выбор палитры — **Вариант Б (Split): светлый hero + тёмные секции**

**Обоснование выбора:**

| Вариант | За | Против | Вердикт |
|---|---|---|---|
| А. Dark везде (как приложение) | Консистентно с приложением. Агрессивно-качковский вайб | Конверсия тёмных лендингов в фитнесе ниже на 15-20% (данные Unbounce 2024). Ощущается "подпольно". Плохо для trust | ❌ |
| **Б. Split: светлый hero + тёмные акцентные секции** | **Высокий trust от светлого hero (как MyFitnessPal, Cronometer). Тёмные секции для "Как работает" и pricing создают контраст и демонстрируют приложение в его родной среде. Лучший конверсионный паттерн fitness-индустрии** | Чуть сложнее в исполнении — нужна аккуратная типографика на переходах | ✅ **ВЫБРАН** |
| В. Light + Navy (как Cronometer) | Максимальный trust, corporate feel | Скучно. Не попадает в качковский вайб. Не выделяется на фоне конкурентов | ❌ |

### 4.2 Цветовая палитра

```scss
// === СВЕТЛЫЕ СЕКЦИИ (Hero, Features, Testimonials, FAQ, Footer) ===
$bg-primary:    #FAFAF7;  // off-white, тёплый (не стерильно-белый)
$bg-secondary:  #FFFFFF;  // карточки
$bg-elevated:   #F4F4EF;  // subtle-секции, разделители
$text-primary:  #0F0F0F;  // почти чёрный
$text-secondary:#5A5A5A;  // вторичный серый
$border:        #E8E8E3;  // тонкие разделители

// === ТЁМНЫЕ СЕКЦИИ (How it works, Pricing, Final CTA) ===
$dark-bg:        #0F0F0F;  // main dark (= фон приложения)
$dark-elevated:  #1A1A1A;  // карточки в тёмных секциях
$dark-text:      #FFFFFF;
$dark-text-soft: #AAAAAA;

// === АКЦЕНТЫ (единые для светлых и тёмных) ===
$accent-primary:   #FF6B00;  // энергичный оранжевый — основной CTA, highlights, цифры
$accent-secondary: #00D9FF;  // холодный голубой — иконки фичей, графики
$accent-success:   #6BCB77;  // зелёный — "добрал норму", галочки в pricing
$accent-warning:   #FFB700;  // золотой — рейтинги (звёзды), badges

// === СИСТЕМНЫЕ ===
$success: #22C55E;
$error:   #EF4444;
```

**Пропорция использования:** 55% светлый фон + 25% тёмный фон + 10% white (карточки) + 10% accents.

### 4.3 Типографика

**Основной стек:**
```css
font-family: 'Inter', -apple-system, BlinkMacSystemFont, 'SF Pro Display', sans-serif;
```

**Display (для Hero h1 и крупных цифр):**
```css
font-family: 'Geist', 'Inter Display', 'Inter', sans-serif;
/* Geist — современный, немного техничный, хорошо читается на мобайле */
```

**Моно (для цифр в сравнительной таблице и метриках):**
```css
font-family: 'JetBrains Mono', 'SF Mono', monospace;
```

**Размеры (fluid-typography через clamp):**

| Элемент | Desktop | Mobile | Weight | Letter-spacing |
|---|---|---|---|---|
| Hero h1 | `clamp(56px, 7vw, 96px)` | 48-56px | 800 (ExtraBold) | -0.03em |
| Section h2 | `clamp(40px, 5vw, 64px)` | 36-40px | 700 (Bold) | -0.02em |
| h3 (feature cards) | 24-28px | 22px | 600 (SemiBold) | -0.01em |
| Body Large (sub-headline) | 20-22px | 18px | 400 | 0 |
| Body | 16-18px | 16px | 400 | 0 |
| Caption / small | 14px | 13px | 500 | 0.01em |
| Mega Numbers ("1800") | 120-160px | 72-96px | 800 | -0.05em |

**Правила:**
- Межстрочный интервал: 1.1 для заголовков, 1.55 для body
- Максимальная ширина параграфа: `max-w-[65ch]` (~720px)
- Никакого `justify` — только `left-aligned`, на мобиле некоторые hero — `center`
- На мобильных h1 **никогда** не меньше 44px (иначе не давит)

### 4.4 Spacing / layout

**Базовый ритм:** кратный 8 (Tailwind-стандарт). Но щедрый, как у Apple/Stripe.

```
Section padding-top/bottom (desktop): 120px (py-30)
Section padding-top/bottom (mobile): 80px (py-20)
Inner gap (между элементами в секции): 48-64px
Card padding: 32px (desktop) / 24px (mobile)
Container max-width: 1280px
Grid gap: 32px
```

### 4.5 Скругления и тени

```css
--radius-sm: 8px;   /* badges, tags */
--radius-md: 12px;  /* buttons */
--radius-lg: 20px;  /* cards */
--radius-xl: 28px;  /* hero mockup, pricing cards */

--shadow-lg: 0 24px 48px rgba(15,15,15,0.12);

```

> Если ты качаешься — затем, что MFP не считает спортпит, не переключает нормы под день тренировки и не знает разницу между "массой" и "поддержкой". Мы умеем всё это, плюс фокус на русской качковской еде. Если худеешь — оставайся на MFP, это не твоё приложение.

> Вся база USDA + Open Food Facts + отдельная база русской качковской еды (куриная грудка разных производителей, творог 0-9%, гречка, рис, протеиновые батончики всех российских брендов, гейнеры, русские БАДы). Борщ и пельмени тоже есть, но фокус — на бодибилдинг-меню.

**4. Работает в России? Оплата в рублях?**
> Да. Сервера на Supabase (EU регион, разрешено 152-ФЗ), оплата через российские платёжные системы в рублях. Без VPN, без танцев с бубном. Приложение в RuStore + пытаемся пройти App Store RU.

**5. А если я не качаюсь, а просто слежу за питанием?**
> Тебе хватит FREE-тарифа. Но если цель "просто поесть правильно" — возьми YAZIO или MFP, они удобнее для твоего кейса. Мы не обижаемся, мы честные.

**6. Что вы делаете с моими данными? Вес, КБЖУ — это чувствительно.**
> Данные хранятся в Supabase (EU) с Row-Level Security — доступ только у тебя. Мы **не продаём** их рекламщикам, **не делимся** с фитнес-клубами, **не используем** для обучения сторонних моделей. В профиле — кнопка "Удалить аккаунт", данные стираются за 24ч.

## 8. Анимации и интерактив

### 8.1 Загрузка страницы
- **Fade-in Hero** (opacity 0 → 1, 400ms ease-out, stagger 80ms между элементами: eyebrow → h1 → sub → CTA → trust-row)
- **Мокап iPhone** влетает справа (translate-x: 40px → 0, 600ms)
- **Плавающие карточки** fade-in с задержкой 800-1200ms

### 8.2 Скролл-анимации (Intersection Observer + Framer Motion)
- Все секции при попадании в viewport → `opacity: 0 → 1, translateY: 24px → 0`, 500ms
- Stagger детей секции: 60ms
- **Не переборщить** — Apple/Stripe-философия, не Awwwards-цирк

### 8.3 Hero: плавающие карточки
```
animation: float 4s ease-in-out infinite;
@keyframes float {
  0%, 100% { transform: translateY(0); }
  50%      { transform: translateY(-8px); }
}
```
Каждая карточка — с разной задержкой (0s, 1s, 2s).

### 8.4 Счётчики (пример: "10K+ качков", "3.5M+ блюд")
- При попадании в viewport — анимация от 0 до целевого числа
- Длительность: 1.5 сек, easing: `cubic-bezier(0.25, 0.46, 0.45, 0.94)`
- Используем `react-countup` или собственный хук

### 8.5 Hover-эффекты
- Feature-карточки: `translateY(-4px)` + shadow-md, 250ms
- CTA кнопки: `translateY(-2px)` + усиление shadow, 200ms
- Логотипы брендов: grayscale → color, 200ms
- Pricing-карточки: hover на "PRO" (выделенный) чуть меньше, чем на остальные (он и так выделен)

### 8.6 Секция "Как работает" — demo-GIF
- Autoplay, loop, muted, `<video>` с preload="metadata" (не metadata, не тяжёлое)
- На мобиле — можно заменить на статический скриншот + play-иконку (tap → открывает fullscreen-видео)

### 8.7 Accordion FAQ
- Плавное раскрытие (height: auto через framer-motion), 300ms ease-out
- Иконка + → × на активном

### 8.8 Carousel скриншотов
- Свайп на touch, arrows на desktop
- Snap-scroll (`scroll-snap-type: x mandatory`)
- Индикатор прогресса снизу (dots)

### 8.9 Sticky nav + progress bar
- Тонкая (2px) оранжевая полоска сверху, показывает scroll progress

### 8.10 Exit-intent модалка (опционально, желательно)
- При движении курсора к кнопке "закрыть вкладку" → модалка с оффером "Подпишись на ранний доступ, получи 30 дней PRO бесплатно при запуске"
- Поле email + CTA
- Срабатывает 1 раз за сессию (cookie)
- **Только desktop** (на мобиле нет exit-intent, используем другие триггеры)

### 8.11 Что НЕ делать
- ❌ Parallax на всю страницу (укачивает на мобиле)
- ❌ Heavy scroll-jacking (bad UX)
- ❌ Video autoplay с звуком
- ❌ Pop-up cookie wall (только лёгкий баннер)
- ❌ Чат-бот в правом углу (для MVP-лендинга — избыточно)

## 14. A/B тесты на будущее

### 14.1 Приоритетные тесты (первые 3 месяца)

| # | Тест | Что меняем | Гипотеза | Метрика победы |
|---|---|---|---|---|
| 1 | **Headline** | "Хватит гадать..." vs "Сфотографируй еду..." vs "200г белка в день..." | Провокация = выше конверсия у качков | hero_cta_click rate |
| 2 | **CTA цвет** | Оранжевый (#FF6B00) vs Зелёный (#6BCB77) | Зелёный = healthy ассоциация, выше на fitness | cta_click rate |
| 3 | **Pricing default** | По умолчанию "мес" vs "год -33%" | Год = выше LTV | pricing_plan_click → паplan "год" |
| 4 | **Hero layout** | 2-колонка (mockup справа) vs single-column (mockup под текстом) | Single может быть читабельней на лэптопах | scroll past hero rate |
| 5 | **Social proof позиция** | После Hero vs после Features | Раньше = больше trust на старте | bounce rate |
| 6 | **Количество тарифов** | 3 (FREE/PRO/PRO+) vs 2 (FREE/PRO) | Меньше выбор = выше конверсия (Hick's law) | subscribe rate |

### 14.2 Правила A/B
- Минимум 2 недели на тест (чтобы захватить разные дни недели)
- Минимум 500 уникальных посетителей на вариант
- Статзначимость 95% (не 90%)
- Одновременно — не больше 2 тестов (чтобы не было взаимовлияния)
- Инструмент: **PostHog Experiments** (free tier хватает)

## 17. Итого

**Лендинг GymFuel AI — это не "ещё один fitness-лендинг".** Это **жёсткий, прямой, ориентированный на качка маркетинговый инструмент**, который:

1. За 5 секунд объясняет: "Это для качков. Фоткаешь еду. AI считает. Спортпит учитывает."
2. Закрывает главные возражения (MFP уже есть, AI неточный, очередная подписка) в первых 3 секциях.
3. Демонстрирует уникальность через сравнительную таблицу — нечестная игра, где GymFuel выигрывает по 10 пунктам из 10.
4. Давит на кнопку CTA минимум 5 раз за скролл.
5. Живёт на светлом фоне (trust, conversion), но ныряет в тёмный (приложение, pricing, финальный CTA — для контраста и драмы).

**Ключевая философия:** копирайтинг — 60% успеха, дизайн — 40%. Если тексты не цепляют ЦА (качков) — никакая картинка не спасёт.

**Главные метрики успеха:**
- Конверсия в скачивание: **3.5-5%**
- Bounce rate: **<50%**
- Средний scroll depth: **>60%**
- Подписка на ранний доступ (до релиза): **15-20%**

**Следующий агент (Агент 6):** на основе этого ТЗ собирает структурированный бриф для дизайнера / генерит Figma-прототип через Stitch / готовит design-ready handoff.
