## 1. Краткое резюме проекта

**GymFuel AI** — AI-счётчик калорий и спортпита для качков (мужчины 20-40, ходят в тренажёрный зал). Приложение (PWA, mobile-first) распознаёт еду по фото, считает КБЖУ под цель (масса/сушка/поддержка) и встраивает спортивное питание (Whey, креатин, BCAA) в план дня. Отдельно — маркетинговый лендинг для конверсии.

**Ключевой дифференциатор в дизайне:** Dark-first + минимализм + жирные цифры. НЕ MyFitnessPal для худеющих домохозяек. НЕ розовая Lifesum. Это Strong + Cal AI, но с спортпитом.

### 2.3 Spacing, Radius, Elevation

**Spacing (4px base):** 4, 8, 12, 16, 20, 24, 32, 40, 48, 64, 80, 96 px

**Border radius:**
- Buttons, inputs: `12px`
- Cards: `16px`
- Sheets, modals: `24px`
- Pills (chips): `999px`

**Elevation (тени в dark mode — очень лёгкие):**
- Level 1: `0 1px 2px rgba(0,0,0,0.3)`
- Level 2: `0 4px 8px rgba(0,0,0,0.4)`
- Level 3 (modal): `0 16px 32px rgba(0,0,0,0.5)`

На лендинге (светлые секции) — стандартные тени: `0 2px 8px rgba(0,0,0,0.08)`

### 2.4 Иконки

- **Библиотека:** Lucide Icons (бесплатная, open-source)
- **Стиль:** Line icons, толщина `1.5px`
- **Размеры:** 16 / 20 / 24 / 32 px
- **Цвет:** `#FFFFFF` (dark) / `#0F0F10` (light) / `#FF6B00` (акцент)

## 3. Приложение GymFuel AI

### 3.1 Design Principles (7)

1. **Dark-first** — фон всегда тёмный, зал тёмный
2. **Numbers over words** — жирные цифры главнее, чем описания
3. **Minimum taps** — до любого действия ≤3 тапа
4. **No BS tone** — никакой мотивационной патоки
5. **Glove-friendly** — кнопки 52px, работает одним пальцем
6. **One thing per screen** — экран решает одну задачу
7. **High contrast** — читаемо при ярком свете в зале

### 3.2 8 MVP-экранов (краткие описания)

| # | Экран | Назначение | Ключевые элементы |
|---|-------|------------|-------------------|
| 1 | **Splash / Login** | Вход в приложение | Логотип + email/Google/Apple OAuth |
| 2 | **Onboarding** (5 шагов) | Сбор данных для расчёта норм | Пол → Вес/рост → Цель → Активность → Спортпит |
| 3 | **Home / Dashboard** | Главный экран, метрики дня | 4 больших числа (Ккал/Белок/Углеводы/Жиры) + кнопки [+ ЕДА] [+ ТРЕНИРОВКА] |
| 4 | **Photo Recognition** | Фото еды → распознавание | Камера → loading → результат → edit → save |
| 5 | **Meal Log** | Дневник питания на день | Список приёмов, редактирование, навигация по дням |
| 6 | **Food Search** | Поиск продукта вручную | Autocomplete + favorites + history |
| 7 | **Profile** | Настройки, цели, параметры | Редактирование веса, цели, спортпита, нормы |
| 8 | **Supplements** | Мой запас спортпита + рекомендации | Список добавок, что пить сегодня, калькулятор |

**Детали wireframe и состояний** (empty / loading / error) → см. `03_ТЗ_дизайн_приложения.md`, раздел 5

## 5. Tone of Voice (единый для app + landing)

### ✅ ДА — говори так

| Контекст | Пример |
|----------|--------|
| CTA | "Записать", "Поехали", "Скачать", "Добить норму" |
| Dashboard | "Белок: 145/200г", "Осталось: 55г" |
| Успех | "✓ Записано", "В дневнике" |
| Пустое состояние | "Тут будут твои приёмы пищи. Жми + чтобы добавить." |
| Ошибка | "Не получилось распознать. Сними ближе и ровнее." |
| Push-уведомление | "Осталось 40г белка. 1 ложка протеина закроет." |
| Вибрация (Vibe) | "Ты заявил цель — 5кг. До неё 62 дня. Сегодня не добрал 40г белка. Иди выпей, или перестань врать себе." |

### ❌ НЕТ — так не говори

| Антипаттерн | Почему |
|-------------|--------|
| "Давай тренироваться с улыбкой! 🌸" | Розовый сироп |
| "Поздравляем! Вы выполнили норму!" | Геймификация надоедает |
| "Выпейте воды" | Патерналистски |
| "Красавчик!", "Огонь!", "Молодчик!" | Фамильярно-глупо |
| "Не расстраивайтесь, что не добрали" | Не нужно утешать |
| "Добавить приём пищи" (в кнопке) | Канцелярит. Лучше "Записать" |
